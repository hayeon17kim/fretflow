## 1. 요약 (Executive Summary)

FretFlow는 React Native/Expo 기반 모바일 기타 학습 앱으로, SM-2 간격 반복 알고리즘을 활용하여 지판 음 위치, 음정(인터벌), 스케일 패턴, 청음 훈련을 4단계 프로그레시브 커리큘럼으로 체계적으로 가르치는 서비스입니다.

### 현재 상태 평가

| 항목 | 점수 | 상태 |
|------|------|------|
| 핵심 알고리즘 (SM-2) | 9/10 | 완전 구현 및 테스트 완료 |
| 커리큘럼 설계 | 8/10 | 4단계 프로그레시브 시스템 완성 |
| UI/UX 완성도 | 6/10 | 기능적이나 온보딩 부재 |
| 기능 완성도 | 5/10 | 필요 기능의 약 60% |
| 수익화 | 0/10 | 매출 인프라 전무 |
| 분석 인프라 | 0/10 | 트래킹/인사이트 전무 |
| 시장 준비도 | 3/10 | 런칭 불가 상태 |

**핵심 결론:** 기술 기반은 탄탄하지만(SM-2 + MMKV 영속성 + 결정적 카드 시스템), 라우팅 로직, 세션 관리, 온보딩, 분석, 수익화의 핵심 공백을 해결해야 시장 진입이 가능합니다.

## 2. 제품 현황 평가

### 2.1 아키텍처 개요

- **프레임워크:** React Native + Expo SDK 54 + Expo Router 6.0
- **상태 관리:** Zustand 5 + MMKV (고성능 네이티브 스토리지)
- **학습 알고리즘:** SM-2 간격 반복 (quality 0-5 스코어링)
- **다국어:** i18next (한국어/영어)
- **오디오:** expo-av, WAV 파일 42개 (5개만 사용 중)

### 2.2 커리큘럼 구조

| 레벨 | 콘텐츠 | 카드 수 | 상태 |
|------|---------|---------|------|
| Lv.1 음 위치 | 지판 위 음 식별 | ~72 | ✅ SM-2 연동 |
| Lv.2 인터벌 | 음정 거리 인식 | ~60 | ✅ SM-2 연동 |
| Lv.3 스케일 패턴 | 스케일 형태 학습 | ~48 | ✅ SM-2 연동 |
| Lv.4 청음 훈련 | 오디오 인식 | ~5/42 | ⚠️ 미활용 |

### 2.3 SM-2 구현 상태

SM-2 알고리즘이 다음 마스터리 기준으로 정확하게 구현되어 있습니다:

- **반복 횟수:** 3회 이상 성공적 복습
- **이지 팩터:** 2.5 이상
- **인터벌:** 7일 이상
- **레벨당 목표 카드:** 60장
- **레벨 잠금해제 기준:** 80% 마스터리

## 3. 기술 아키텍처 리뷰

### 3.1 강점

1. **MMKV 스토리지:** AsyncStorage 대비 10배 빠른 고성능 네이티브 영속성, 앱 크래시 시에도 데이터 손실 제로
2. **결정적 카드 ID:** `note-C-3-5`, `interval-E-A-P5` 형식으로 중복 카드 제거 및 신뢰할 수 있는 추적 가능
3. **Zustand 스토어 설계:** `useAppStore`와 `useSpacedRepetition` 훅으로 관심사 분리가 깔끔
4. **접근성:** 다크 테마, 적절한 대비율, 햅틱 피드백 지원, RTL 레이아웃 대응
5. **TypeScript:** 코드베이스 전체에 완전한 타입 안전성

### 3.2 기술 리스크

1. **죽은 Supabase 코드:** MMKV로 완전 이전했음에도 Supabase import 참조가 코드에 남아 있음 — 신규 개발자에게 혼란 유발
2. **에러 바운더리 없음:** 처리되지 않은 에러 시 복구 메커니즘 없이 앱이 크래시됨
3. **진행률 계산 불일치:** `useHomeScreenStats`는 `getCardCount(level)/60`(전체 카드), `useSpacedRepetition.getLevelProgress`는 마스터된 카드/60 사용 — 사용자에게 상충되는 숫자 표시
4. **오프라인 처리 없음:** 네트워크 상태 감지나 우아한 성능저하 미구현
5. **오디오 로딩:** 모든 WAV 파일이 동기적으로 로드됨 — 저사양 기기에서 시작 지연 가능성

## 5. 핵심 이슈 및 공백

### 5.1 P0 — 런칭 전 반드시 수정

#### 이슈 1: 스마트 리뷰 라우팅 고장

**위치:** `src/app/(tabs)/index.tsx`, 36번째 줄
**문제:** `handleStartReview`가 어떤 레벨에 복습할 카드가 있든 상관없이 항상 `/quiz/note`로 이동
**영향:** Lv.2-4에 복습할 카드가 있는 사용자가 Lv.1 음 복습만 강제됨. 전체 프로그레시브 학습 모델이 깨지며 사용자 불만과 이탈 유발
**해결:** 모든 잠금해제된 레벨의 복습 카드를 확인하고, 가장 복습 카드가 많은 레벨로 라우팅하는 우선순위 기반 라우팅 구현 또는 레벨 선택 화면 제공

#### 이슈 2: 세션 크기 옵션이 장식용

**위치:** `src/app/(tabs)/practice.tsx`
**문제:** Quick(10), Focus(25), Deep(50) 세션 버튼이 모두 하드코딩된 `SESSION_SIZE = 10`으로 동일한 퀴즈 실행
**영향:** UI 내 허위 기능. "Deep 50"을 선택한 사용자가 10장만 받음. 신뢰 파괴
**해결:** 세션 크기를 라우트 파라미터로 전달하고 퀴즈 로직에서 반영

#### 이슈 3: 온보딩 플로우 없음

**문제:** 신규 사용자가 FretFlow가 무엇을 하는지, 간격 반복이 어떻게 작동하는지, 어떻게 시작하는지에 대한 맥락 없이 홈 화면에 바로 도착
**영향:** 기타 학습 앱은 Day-1 이탈률이 70% 이상. 온보딩 없이는 대다수 신규 사용자를 즉시 잃게 됨
**해결:** 3-5 화면 온보딩 플로우 구현: 앱 가치 제안, 빠른 실력 평가, 일일 목표 설정, 첫 카드 튜토리얼

#### 이슈 4: 분석 인프라 제로

**문제:** 이벤트 트래킹, 세션 기록, 퍼널 분석 없음
**영향:** 리텐션 측정 불가, 이탈 지점 파악 불가, 데이터 기반 의사결정 불가. 완전히 맹목적 운영
**해결:** Mixpanel 또는 Amplitude 통합. 추적 항목: session_start, card_reviewed, level_unlocked, streak_achieved, subscription_viewed

#### 이슈 5: 수익화 모델 없음

**문제:** 구독 시스템, 페이월, 프리미엄 기능 정의 없음
**영향:** 매출 창출 불가. 경쟁사(Yousician $9.99/월, Fender Play $9.99/월)는 이미 지불 의향을 확보 중
**해결:** 3단계 프리미엄 모델 구현 (7장 참조)

### 5.2 P1 — 첫 달 내 수정

#### 이슈 6: 청음 훈련 미활용

42개 WAV 오디오 파일 중 5개만 Lv.4에서 사용. 오디오 콘텐츠의 88%가 낭비됨. 42개 전체로 확장 시 Lv.4가 훨씬 가치 있는 기능으로 변모

#### 이슈 7: 믹스 모드가 스텁

Practice의 Mix 모드 버튼이 존재하지만 `/quiz/note`로만 라우팅. 크로스 레벨 혼합 복습은 경쟁사에 없는 핵심 차별화 요소

#### 이슈 8: 푸시 알림 없음

간격 반복 앱은 일일 복귀율에 생사가 달림. 복습 카드 알림 없이는 Day 3 이후 리텐션 붕괴

#### 이슈 9: 일일 목표 시스템 없음

스트릭 추적, 일일 목표, 성취 시스템 없음. 학습 앱의 기본 요소

#### 이슈 10: 진행률 계산 불일치

홈 화면과 마스터리 화면이 같은 레벨에 대해 다른 계산 방식으로 서로 다른 진행률 표시

## 8. Go-to-Market 로드맵

### Phase 1: 기반 수정 (1-4주차)

**목표:** 모든 P0 이슈 수정, 성장을 위한 기술 기반 확립

- 스마트 리뷰 라우팅을 전 레벨 대상으로 수정
- 실제 세션 크기 옵션(10/25/50) 구현
- 에러 바운더리 및 크래시 복구 추가
- 죽은 Supabase 코드 제거
- 진행률 계산 불일치 수정
- **마일스톤:** 크리티컬 버그 0건의 내부 알파

### Phase 2: 성장 인프라 (5-8주차)

**목표:** 리텐션 및 분석 시스템 구축

- 온보딩 플로우 구현 (3-5 화면)
- 분석 도구 통합 (Mixpanel/Amplitude)
- 복습 카드 푸시 알림 추가
- 일일 목표 및 스트릭 시스템 구현
- 청음 훈련 42개 사운드 전체 확장
- **마일스톤:** 50-100명 테스터 클로즈드 베타

### Phase 3: 수익화 (9-12주차)

**목표:** 매출 모델 구현 및 검증

- 구독 시스템 구축 (RevenueCat)
- 무료/프로 기능 게이팅 구현
- 페이월 화면 디자인 및 테스트
- 설정에 구독 관리 추가
- 가격 및 페이월 배치 A/B 테스트
- **마일스톤:** 매출이 발생하는 베타

### Phase 4: 완성 및 런칭 준비 (13-16주차)

**목표:** 앱스토어 제출 준비

- 크로스 레벨 믹스 모드 구현
- 소셜 기능 (리더보드, 공유)
- 앱스토어 최적화 (ASO) — 스크린샷, 설명, 키워드
- 성능 최적화 및 디바이스 테스트
- 접근성 감사
- **마일스톤:** 앱스토어 제출

### Phase 5: 런칭 및 성장 (17-24주차)

**목표:** 시장 진입 및 사용자 확보

## 12. 최종 권고

FretFlow는 **강력한 기술 기반**과 잘 구축되지 않은 지판 이론 니치에서의 **명확한 시장 기회**를 보유하고 있습니다. SM-2 알고리즘, 프로그레시브 커리큘럼, MMKV 영속성은 잘 구현되어 있습니다.

그러나 **제품은 런칭 준비가 되지 않았습니다.** 5개 P0 이슈(특히 고장난 라우팅과 미구현 수익화)를 먼저 해결해야 합니다.

### 권고 경로:

1. **1-4주차:** 전체 P0 버그 수정 + 온보딩 + 분석 → 내부 알파
2. **5-8주차:** 리텐션 기능 (알림, 스트릭) + 콘텐츠 확장 → 클로즈드 베타
3. **9-12주차:** 수익화 (RevenueCat + 페이월) + A/B 테스트 → 매출 베타
4. **13-16주차:** 완성 + ASO + 믹스 모드 → 앱스토어 제출
5. **17-24주차:** 한국 소프트 런칭 → 커뮤니티 아웃리치 → 스케일

**이 6개월 로드맵을 규율 있게 실행하면, FretFlow는 첫해에 현실적으로 1만-5만 사용자와 ₩1.7억-₩6.7억 연간 반복 매출(ARR)을 달성할 수 있습니다.**
