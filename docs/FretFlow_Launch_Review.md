# FretFlow Product Review & Launch Readiness

**Senior PM/BD/PD 평가 · 2026년 2월 17일**

---

## Executive Summary

FretFlow는 기타 프렛보드 암기 학습을 위한 React Native 앱으로, SM-2 알고리즘 기반의 간격 반복 시스템을 핵심으로 합니다. 코드베이스 전반적으로 MVP 수준의 완성도를 보이며, 학습 트랙 4개 + Mix 모드의 콘텐츠 구성은 탄탄합니다.

**핵심 결론:** 학습 엔진의 기술적 기반은 훌륭하지만, 유저가 지속적으로 쓸 이유(즉, Retention을 만드는 감정적 경험)가 부족합니다. "잘 만든 플래시카드 앱"에서 "매일 열고 싶은 기타 훈련 파트너"로 진화하려면, 론칭 전 4주간 리텐션 레이어를 집중적으로 보강해야 합니다.

## 종합 평가 스코어카드

"이 앱을 내일도 열고 싶을까?"를 기준으로 평가했습니다.

| 평가 영역 | 점수 | 코멘트 |
|-----------|------|--------|
| 학습 엔진 (SM-2) | 9/10 | SM-2 구현 정확하고, 반응시간 반영도 좋음 |
| 콘텐츠 구성 | 8/10 | 4개 트랙 + Mix 모드 + 티어 시스템으로 볼륨 충분 |

| 사운드 / 피드백 | 5/10 | 오디오 재생 있으나, 정답/오답 시 피드백이 약함 |
| 모니타이제이션 준비 | 1/10 | 수익화 구조 전무. 유료 전환 시나리오 미반영 |

## 솔직한 분석: 유저가 정말 쓸까?

### 현재 강점 (진짜 잘한 것)

**1. SM-2 알고리즘 구현이 정확하고 실용적입니다.** 반응시간까지 반영하는 quality 매핑(3초 미만 = 5, 6초 미만 = 4, 그 이상 = 3)은 Anki보다 더 세밀하고, 기타 학습 도메인에 적합합니다.

**2. 4개 트랙 + Mix 모드의 콘텐츠 구성이 좋습니다.** Note Position → Intervals → Scales → Ear Training의 순서가 음악 이론 학습 경로와 일치하고, 티어 시스템으로 점진적 난이도 조절이 자연스럽습니다.

**3. 기술 스택이 실용적입니다.** MMKV를 사용한 빠른 로컬 저장, Zustand의 간결한 상태 관리, Expo Router로 정갈한 네비게이션. 오버엔지니어링 없이 필요한 것만 잘 선택했습니다.

### 핵심 문제 (유저가 이탈할 이유)

#### 문제 1: "아하 모먼트"가 너무 늦게 옵니다

현재 온보딩은 4단계(소개 → 미니퀴즈 → 결과 → 목표설정)로 구성되어 있지만, 유저가 "오, 이게 나한테 도움이 되겠다!"라고 느끼는 순간이 명확하지 않습니다. 미니퀴즈에서 틀린 문제를 맞춰도 "그래서 뭐?"라는 반응이 나올 수 있습니다.

비교: Duolingo는 첫 30초 안에 "너 이미 이걸 알고 있어!"라는 성취감을 줍니다.

#### 문제 2: 매일 돌아올 감정적 동기가 부족합니다

현재의 리텐션 장치는 스트릭(연속 학습일)과 배지(5단계)뿐입니다. 그런데 스트릭은 "끊어지면 의미 없음" 심리를 만들고, 배지는 한번 따면 끝입니다. 유저가 2주차에 사용을 멈춘다면, 돌아올 이유가 없습니다.

Duolingo의 핵심은 알고리즘이 아니라 "XP, 리그, 친구 경쟁, 하트" 같은 감정 레이어입니다.

#### 문제 3: 퀴즈 종료 시 UX가 약합니다

퀴즈 중간에 나가면 진행 상황이 모두 사라집니다. 완료 화면의 동기부여도 "성적 요약 + 내일 복습 카드 수" 정도로, 유저가 자신의 성장을 체감하기 어렵습니다. "어제보다 3초 빨라졌어요!" 같은 피드백이 없습니다.

#### 문제 4: 사운드가 학습 경험을 완성하지 못합니다

기타 학습 앱에서 사운드는 핵심입니다. 정답일 때 해당 음을 들려주는 것만으로는 부족합니다. 오답 시 정답음을 들려주면서 시각적으로 위치를 보여주는 멀티모달 피드백이 필요합니다. Fender Play나 Yousician은 모든 상호작용에 사운드를 결합합니다.

#### 문제 5: 데이터 없이 비행하고 있습니다

애널리틱스가 전무합니다. 어떤 트랙이 인기 있는지, 어디서 이탈하는지, 어떤 문제가 어려운지 전혀 모릅니다. 론칭 후 제품 개선의 방향을 잡을 수 없습니다.

| | Fretboard Learn | NoteTrainer | FretFlow |
|---|---|---|---|
| 간격반복 | ✗ | △ | **◎ SM-2 정확 구현** |
| 트랙 수 | 1개 | 2개 | **4개 + Mix** |
| 티어 시스템 | ✗ | ✗ | **◎ 4단계** |
| 게이미피케이션 | △ | △ | **△ 부족** |

**FretFlow의 차별화 포인트:** SM-2 기반 간격반복 + 4개 트랙 + 티어 시스템의 조합은 독보적입니다. 하지만 이 장점은 기술적 장점이지, 유저가 체감하는 장점이 아닙니다. 유저는 "SM-2 알고리즘"을 모릅니다. "어제보다 더 잘 맞춘다"는 체감으로 번역해야 합니다.

## 타겟 유저 분석

**가장 잘 맞는 유저:** 기타 치고 1-3년차, 코드를 보고 연주하지만 프렛보드 전체를 자유롭게 사용하지 못하는 중급자

초보자에게는 "프렛보드 암기"의 필요성을 모르고, 상급자에게는 이미 알고 있는 내용입니다. 이 중급자 타겟은 적절하지만, 이들은 이미 연습 루틴이 있어서 새 앱을 그 루틴에 끼워넣기 어렵습니다. 기존 연습 전후 5분씩 FretFlow를 하는 "보조 훈련 도구" 포지션이 가장 현실적입니다.

**한국 시장 고려사항:** 한국어/영어 i18n이 구현되어 있어 한국 시장 진입이 가능합니다. 한국 기타 인구는 약 200만 명으로 추정되며, 이 중 프렛보드 학습에 관심 있는 중급자는 약 20-40만 명일 것입니다.

## 4주 론칭 액션 플랜

### P0: 론칭 전 필수 (1-2주차)

#### P0-1. Developer Mode 숨기기
- **현상:** Settings 화면에 개발자 모드(마스터리 카드 시뮬레이터, 전체 리셋)가 일반 유저에게 노출됨
- **액션:** `settings.tsx`에서 `__DEV__` 플래그 또는 숨겨진 제스처(로고 7회 탭 등)로 접근 제한
- **공수:** 0.5일

#### P0-2. 에러 바운더리 추가
- **현상:** 에러 바운더리가 없어서 비정상적 상황에서 앱이 크래시됨
- **액션:** 화면별 ErrorBoundary 컴포넌트 + 폴백 UI("문제가 발생했습니다. 다시 시도해주세요") 추가. 최소한 `_layout.tsx` 레벨에서 전체 감싸기
- **공수:** 1일

#### P0-3. 퀴즈 중단 확인 UX
- **현상:** 뒤로가기로 퀴즈를 나가면 진행 상황이 모두 소실됨
- **액션:** 뒤로가기 시 "정말 나가시겠습니까? 진행 상황이 저장되지 않습니다" 확인 모달 + 이미 완료한 카드는 부분 저장하는 로직 추가
- **공수:** 1일

#### P0-4. 기본 애널리틱스 세팅
- **현상:** 애널리틱스 전무. 유저 행동 데이터 수집 불가
- **액션:** 최소한 아래 이벤트 추적 세팅 (Amplitude, Mixpanel, 또는 PostHog 중 택 1)
  - `app_open` / `app_close`
  - `onboarding_start` / `onboarding_complete` / `onboarding_skip`
  - `session_start(track, sessionSize)` / `session_complete(track, accuracy, duration)`
  - `track_selected(trackId)` / `tier_unlocked(trackId, tier)`
  - `streak_achieved(days)` / `badge_earned(track, level)`
  - `daily_goal_achieved` / `notification_opened`
- **공수:** 1-2일

#### P0-5. Supabase 데드코드 제거
- **현상:** 의존성에 `@supabase/supabase-js`가 있지만 실제 사용되지 않음. 번들 크기만 늘림
- **액션:** `package.json`에서 제거 + 관련 import/config 파일 정리
- **공수:** 0.5일

#### P0-6. Mix 모드 completion 트래킹 버그 수정
- **현상:** `mix.tsx`에서 completion으로 라우팅 시 `trackId`가 항상 `'note'`로 고정됨
- **액션:** `trackId: 'mix'`로 변경하거나, completion 화면에서 mix 모드용 별도 표시 처리
- **공수:** 0.5일

### P1: 론칭 첫주 해결 (2-3주차) — 리텐션 핵심

#### P1-1. 온보딩 Aha 모먼트 재설계
- **현상:** 미니퀴즈 후 "그래서 뭐?" 반응 가능
- **액션:**
  - 미니퀴즈를 "진단 테스트"로 리프레이밍: "당신의 프렛보드 실력을 확인해볼까요?"
  - 결과 화면에서 약점을 시각화: "12프렛 이상 영역이 약하네요! FretFlow가 여기를 집중 훈련해드릴게요"
  - 첫 세션 후 즉각 피드백: "5문제 중 3개를 맞혔어요! 내일이면 5개 다 맞출 수 있어요" (SM-2가 보장하는 복습 스케줄을 근거로)
- **공수:** 2-3일

#### P1-2. 세션 완료 화면 강화 — 성장 체감
- **현상:** 완료 화면이 "성적 요약 + 내일 복습 카드 수"로 밋밋함
- **액션:**
  - **응답시간 트렌드 표시:** "이번 세션 평균 2.3초 → 지난 세션 3.1초. 0.8초 빨라졌어요!"
  - **마스터리 진척도 바:** "Note Position 트랙: 24/60 마스터 (40%)"를 시각적 프로그레스 바로 표시
  - **"어제 vs 오늘" 비교 카드:** 틀렸다가 오늘 맞춘 카드를 하이라이트 ("C#3을 드디어 마스터했어요!")
  - **동기부여 메시지 고도화:** 정확도별 차등 메시지 + SM-2 데이터 기반 개인화 ("내일 복습하면 12개 카드가 장기 기억으로 넘어갑니다")
- **공수:** 2-3일

#### P1-3. 정답/오답 사운드 피드백 개선
- **현상:** 기타 학습 앱에서 사운드는 핵심인데, 현재 피드백이 약함
- **액션:**
  - **정답 시:** 해당 음 재생 + 짧은 성공 사운드 + 햅틱 피드백
  - **오답 시:** 정답음 재생 + 프렛보드 위에서 정답 위치 하이라이트 애니메이션 (1.5초간)
  - **연속 정답 시:** 콤보 사운드 이펙트 (3연속, 5연속, 10연속)
- **공수:** 2일

#### P1-4. "오늘의 복습" 푸시 알림 개인화
- **현상:** 알림이 설정되어 있지만 내용이 일반적
- **액션:**
  - 알림 문구에 구체적 숫자 포함: "오늘 복습할 카드 8장이 기다리고 있어요! (약 3분)"
  - 스트릭 위험 알림: "5일 연속 학습 중! 오늘도 이어가세요"
  - 마일스톤 알림: "3개만 더 마스터하면 Tier 2 잠금해제!"
- **공수:** 1일

#### P1-5. 퀴즈 중 진행률 & 콤보 표시
- **현상:** 퀴즈 중 몇 번째 문제인지, 현재 정답률이 어떤지 실시간 피드백 없음
- **액션:**
  - 상단에 "12/25" 형태의 프로그레스 바 (이미 QuizHeader에 일부 있으나 강화)
  - 연속 정답 수 카운터 + 시각적 강조 (불꽃 아이콘 등)
  - 세션 중간 마일스톤: "절반 완료! 현재 정확도 85%"
- **공수:** 1-2일

### P2: 론칭 후 반복 개선 (4주차+)

#### P2-1. 리텐션 레이어 구축
- **주간 목표 시스템:** 일일 목표 외에 주간 목표(이번 주 5일 학습, 새 카드 20개 마스터 등) 추가
- **트랙별 레벨 시스템:** 현재 배지 5단계를 XP 기반 레벨 시스템으로 확장 (레벨업 시 애니메이션 + 새 콘텐츠 잠금해제)
- **"복귀 환영" 메시지:** 3일 이상 미접속 후 돌아오면 "돌아오셨네요! 복습할 카드 23장이 쌓여있어요. 오늘 10장만 해볼까요?" (부담 낮추기)
- **마스터리 맵:** 프렛보드 전체를 시각적으로 보여주고, 마스터한 위치를 색으로 채워가는 "지도 완성" 게임 요소

#### P2-2. 모니타이제이션 기반 설계
- **무료 티어:** 하루 10카드, Note Position 트랙만 전체 접근, 나머지 트랙 Tier 1만
- **프로 티어:** 무제한 카드 + 전체 트랙/티어 + 상세 통계 + Mix 모드
- **가격:** 월 $4.99 / 연 $29.99 (기타 앱 시장 기준 적정)
- **전환 포인트:** 유저가 Note Position Tier 2를 클리어하고 Intervals에 관심을 보일 때 → "프로로 업그레이드하면 Intervals, Scales, Ear Training까지 마스터할 수 있어요"

#### P2-3. 앱 스토어 최적화 (ASO)
- 스크린샷 5장 준비 (프렛보드 UI, 학습 진척도, 세션 완료, 배지 시스템, 다크모드)
- 키워드: guitar fretboard, learn guitar notes, 기타 지판, 기타 음이름, spaced repetition guitar
- A/B 테스트 가능한 설명문 준비

#### P2-4. 테스트 코드 작성
- 현재 테스트가 전무함
- 최우선: `sm2.ts` 유닛 테스트 (알고리즘 정확성), `useSpacedRepetition.ts`의 카드 생성/복습 로직
- 그 다음: 온보딩 플로우 E2E 테스트, 퀴즈 세션 통합 테스트

#### P2-5. Weak Cards 학습 경로 추가
- **현상:** Mastery 화면에서 Weak Cards(easeFactor < 2.0)를 보여주지만, 이를 집중 학습하는 경로가 없음
- **액션:** "약점 집중 모드" 추가 — Weak Cards만 모아서 세션을 구성하는 퀵 액션 버튼

## 최종 판단

> **"잘 만든 학습 엔진을 가진 앱이지만, 아직 제품은 아닙니다."**

학습 알고리즘과 콘텐츠 구성은 9/10 수준이지만, 이것은 "엔진"일 뿐입니다. 유저가 경험하는 것은 엔진이 아니라 "오늘도 5분만 할까?" 하는 감정입니다.

**론칭 가능 여부:** P0 작업을 완료하면 론칭은 가능합니다. 하지만 "성공적인 론칭"이 되려면, P1까지 함께 해결하고 론칭 직후 리텐션 데이터를 보며 P2를 빠르게 실행해야 합니다.

**추천 론칭 전략:** "조기 접근 프로그램"으로 한국 기타 커뮤니티(에브리기타, 클래식기타배움터 등) 100-200명에게 먼저 배포하여 리텐션 데이터를 모으고, 그 데이터 기반으로 P2를 개선한 후 정식 론칭하는 것이 가장 안전합니다.
